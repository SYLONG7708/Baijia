如何取得 Oracle VM 公網 IP、SSH 使用者名稱、私鑰檔案路徑

你要把 Baijia Pro 上傳到 Oracle 並啟動 24 小時紀錄，我需要能從這台電腦連進你的 Oracle Ubuntu VM。 這頁只教你找出連線資料，不會要求你公開私鑰內容。

私鑰檔案內容不能貼到聊天、GitHub、Line、Telegram。你只需要告訴我私鑰「在你電腦的檔案路徑」， 例如 C:\Users\Administrator\Downloads\ssh-key-2026-05-25.key。

最後要回覆給我的格式

完成下面步驟後，把這三行回覆給我：

Oracle VM 公網 IP:
SSH 使用者名稱: ubuntu
SSH 私鑰檔案路徑:
如果你的 VM 是 Ubuntu，使用者名稱通常就是 ubuntu。如果你選的是 Oracle Linux，才通常是 opc。 Baijia Pro 建議使用 Ubuntu，所以預設填 ubuntu。

第一部分：確認你已建立 Ubuntu VM

系統

Oracle VM Image 必須是 Ubuntu。建議 Ubuntu 24.04 或 22.04。

Shape

Always Free 建議 VM.Standard.A1.Flex，先用 2 OCPU / 12 GB RAM。

網路

VM 必須有 Public IPv4，且稍後要開放 TCP 4173。

如果你還沒建立 VM，先打開 Oracle 註冊與建 VM 教學，完成後再回到本頁。

第二部分：找到 Oracle VM 公網 IP

登入 Oracle Cloud Console

打開 https://cloud.oracle.com/。如果它要求 region，選你目前的 ap-tokyo-1。

進入 Instances

左上角選單點開，選 Compute，再選 Instances。

點你的 VM 名稱

例如 baijia-pro-24h。如果列表看不到，確認左側的 Compartment 是否選到正確位置。

複製 Public IP address

在 Instance 詳細頁找到 Instance access 或網路資訊區塊，複製 Public IP address。

正確格式看起來像：

129.xxx.xxx.xxx
不要複製 Private IP。Private IP 通常是 10.x.x.x，只能在 Oracle 內部網路使用，我這台電腦連不上。

第三部分：確認 SSH 使用者名稱

你建立 VM 時選的系統 SSH 使用者名稱 Baijia 建議
Ubuntu ubuntu 使用這個
Oracle Linux opc 不建議，除非你已經選了 Oracle Linux

如果你按照我的 Oracle 教學建立 VM，這一欄請回覆：

SSH 使用者名稱: ubuntu

第四部分：找到 SSH 私鑰檔案路徑

回想你建立 VM 時選的 SSH Key 方式

如果你點的是 Oracle 的 Generate a key pair for me，瀏覽器通常會下載一個 .key 或 .pem 檔案到 Downloads。

在 Windows 下載資料夾找檔案

打開：

C:\Users\Administrator\Downloads

找類似：

ssh-key-2026-05-25.key
ssh-key-xxxx.pem
如果你自己用 ssh-keygen 建立

私鑰通常在：

C:\Users\Administrator\.ssh\id_ed25519

公鑰才是 id_ed25519.pub。我要的是沒有 .pub 的私鑰檔案路徑。

用 PowerShell 搜尋私鑰檔案

如果你找不到，打開 PowerShell 貼上：

Get-ChildItem "$env:USERPROFILE\Downloads","$env:USERPROFILE\.ssh" -Recurse -Include *.key,*.pem,id_ed25519 -ErrorAction SilentlyContinue | Select-Object FullName,LastWriteTime

把顯示出來的 FullName 檔案路徑回覆給我即可。

只回覆路徑，不要打開檔案，不要複製裡面的 -----BEGIN OPENSSH PRIVATE KEY----- 內容。

第五部分：自己先測試 SSH 能不能連

把下面命令裡的 IP 與私鑰路徑換成你的資料：

ssh -i "C:\Users\Administrator\Downloads\ssh-key-xxxx.key" ubuntu@你的Oracle公網IP

第一次會問：

Are you sure you want to continue connecting?

輸入：

yes

如果成功，你會看到類似：

ubuntu@baijia-pro-24h:~$
只要你自己能 SSH 成功，我就能用同一組 IP、使用者名稱、私鑰路徑幫你部署 Baijia Pro。

第六部分：開放 TCP 4173 給 Baijia Pro 網頁

SSH 只需要 22，但 Baijia Pro 首頁需要 4173。請在 Oracle Console 開放：

回到 Instance 詳細頁

點 VM 的 Primary VNIC，再點 Subnet。

進入 Security List 或 Network Security Group

如果 VM 使用 Security List，點 Default Security List。如果你建立時用了 NSG，就進入對應的 Network Security Group。

新增 Ingress Rule
Source CIDR0.0.0.0/0，或只填你的固定 IP 比較安全
IP ProtocolTCP
Source Port Range留空
Destination Port Range4173
DescriptionBaijia Pro dashboard
如果你不開 4173，我仍可用 SSH 部署，但你在瀏覽器打開 http://Oracle公網IP:4173/ 會失敗。

第七部分：把三項資料回覆給我

請照這個格式回覆：

Oracle VM 公網 IP: 129.xxx.xxx.xxx
SSH 使用者名稱: ubuntu
SSH 私鑰檔案路徑: C:\Users\Administrator\Downloads\ssh-key-xxxx.key

我收到後會直接執行：

  • SSH 連線測試。
  • 安裝 Baijia Pro 到 Oracle Ubuntu。
  • 上傳目前本機所有百家樂資料備份。
  • 還原 SQLite 紀錄與訓練資料。
  • 設定 systemd 24 小時常駐。
  • 檢查 http://Oracle公網IP:4173/ 是否正常。

常見錯誤

錯誤 原因 處理方式
Permission denied (publickey) 私鑰錯、使用者名稱錯，或 VM 建立時貼的公鑰不是這把私鑰對應的公鑰。 Ubuntu 用 ubuntu；確認私鑰是建立 VM 時下載的那一個。
Connection timed out IP 錯、VM 沒開、或 Oracle 沒開放 SSH 22。 確認 Public IP，Instance 狀態是 Running，Security Rule 有 TCP 22。
WARNING: UNPROTECTED PRIVATE KEY FILE Windows 私鑰權限太開。 PowerShell 執行：
icacls "C:\路徑\ssh-key.key" /inheritance:r
icacls "C:\路徑\ssh-key.key" /grant:r "$env:USERNAME:(R)"
沒有 Public IP 建立 VM 時沒有勾 Assign public IPv4 address。 到 VNIC 指派 Public IPv4，或重建 VM 並勾選 Public IP。
